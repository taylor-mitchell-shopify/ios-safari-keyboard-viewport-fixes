This page has a fixed header. The iframe below contains an input that uses the decoy strategy to prevent viewport shift on iOS Safari.

How it works:

1. The iframe contains both a decoy input (positioned high) and the real input

2. When user taps the real input, we intercept and focus the decoy instead

3. The decoy is high enough that Safari won't shift the viewport

4. After 100ms, we swap focus back to the real input
